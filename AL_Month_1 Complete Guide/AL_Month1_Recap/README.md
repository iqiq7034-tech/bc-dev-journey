# 🚀 30 Days of AL Language — Month 1 Reference Guide

### #90DaysOfBCDevJourney | Microsoft Dynamics 365 Business Central

This repository contains a **complete Month 1 reference guide** for learning **AL Language** used in Microsoft Dynamics 365 Business Central.

It covers **30 days of structured learning** with practical explanations including:

- ✅ What
- ✅ When
- ✅ Why
- ✅ Where
- ✅ Syntax & Examples

---

## 📚 What You'll Learn

### 📦 Tables (Days 1–8)

- DataClassification (GDPR compliance)
- Field validation (NotBlank, Min/Max, InitValue)
- TableRelation (foreign keys)
- FlowFields & CalcFormula
- Keys & performance optimization
- Table Triggers
- Bulk operations (DeleteAll)

---

### 📄 Pages (Days 9–14)

- Page Types (Card, List, Document)
- Layout controls (group vs repeater)
- DrillDown & SubPages
- Promoted Actions (UI optimization)
- Styling (Favorable, Unfavorable, etc.)
- Page Triggers & Validation

---

### 🔧 Extensions (Days 15–17)

- tableextension & pageextension
- Layout positioning (addafter, addbefore)
- Modifying existing UI elements

---

### ⚙️ Codeunits (Days 18–20)

- Local vs Public procedures
- `var` parameters (pass by reference)
- Return types & exit()
- Notification vs Message

---

### 📊 Reports & Queries (Days 21–22)

- DataItems & DataItemLink (joins)
- CalcFields in reports
- Query objects for fast aggregation

---

### 📁 Record Operations (Day 23)

- Get / FindFirst / FindSet
- Filtering (SetRange, SetFilter)
- CRUD operations (Insert, Modify, Delete)
- Performance tips (IsEmpty vs Count)

---

### ⚡ Events (Day 24)

- Integration Events
- Event Subscribers
- IsHandled pattern

---

### 🔄 Data Formats (Day 25)

- JSON (APIs, web integrations)
- XML (EDI, banking, government formats)

---

## 🧠 Bonus Content

- Key comparisons (Triggers vs Procedures, ENUM vs Option, etc.)
- Quick revision summary (all 30 topics)
- Advanced concepts used by **senior AL developers**

---

## 💡 Key Principles

- 📌 Business logic → Codeunits
- 📌 UI → Pages
- 📌 Data rules → Tables
- 📌 Always think **performance + scalability**
